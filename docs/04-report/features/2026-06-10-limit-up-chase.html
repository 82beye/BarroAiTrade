1배경 / 목적

운영 시스템은 그동안 급등 추격 손실을 겪어, 데몬·봇에 급등 진입을 막는 글로벌 가드를 겹겹이 쌓아왔다(_MAX_FLU_RATE=30, P10 시초가폭등 차단 20%, 고점인접 차단 1.5%). 이는 보수 전략(f/sf/gold/swing)엔 옳지만, 상한가 모멘텀을 의도적으로 추격하는 전략과는 정반대다.

한편 "캡상승 수익극대화"(상한가 홀딩 → 수익잠금 → 고점되돌림 청산, 익일 시가갭 부분익절)는 이미 supertrend_auto_trader.py의 RUNNER 기능으로 구현돼 있으나 비활성이었다. 즉 부족한 것은 상따 '진입' 로직이다.

목표 — 보수 전략·글로벌 가드를 전혀 건드리지 않으면서, RUNNER 청산을 재사용하는 분리된·env토글·기본 OFF·dry_run 우선 상따 트레이더를 추가한다.

3진입 로직 AND 조건

[A] 후보 소싱 picker(min_flu_rate↑) → 급등 주도주 (데몬 글로벌가드와 무관) │ [B] 모멘텀 밴드 20% ≤ 등락률 ≤ 27% (+30% 락 추격은 차단) │ [C] 호가 매수벽 ① 매수1호가 ≥ 상한가가격×(1−near%) (ka10004) ② 매수1호가 잔량 ≥ 5만주 │ ③ top-N 매수/매도 잔량 비율 ≥ 3.0 (매도 전무=락임박→통과) │ [D] 시간 게이트 09:05 ~ 14:00 (장막판 추격 차단) ▼ ✅ 진입 (strategy=limit_up_chase, single_tranche)

호가 미가용/조회 실패 시 보수적으로 진입 보류. 거래대금·거래량 급증은 picker 3-factor 랭킹이 보장하므로 추가 TR 호출 없음(429 절감).

4청산 로직 RUNNER 재사용 + 오버나잇

매 사이클 우선순위

  1. 익일 시가갭 부분익절 (오버나잇 한정)
  2. EOD 강제청산 (daily 모드, 15:15)
  3. 트레일청산 고점종가 − 2×ATR
  4. 하드손절 진입가 대비 −4%
  5. RUNNER 상한가권=홀딩, 수익잠금·되돌림 시 청산
  6. 고정 익절 runner 트리거 전 보완

오버나잇 모드 LIMIT_UP_OVERNIGHT_MODE

daily(기본): 상한가 락이라도 15:15 자체 강제청산 → 당일 정리.

overnight: 상한가 락 종목 익일 보유 → 개장 초 시가갭에서 gap_partial(절반) 확정, 잔량은 RUNNER로 런(점상한가 연속 노림).

5포지션 소유권 / 충돌 방지

지점메커니즘
데몬 신규매수 회피상따가 active_positions.json 등록 → 데몬 excluded에 포함 → 같은 종목 미매수
상호 배타 청산상따는 startswith("limit_up_chase")만, supertrend는 "supertrend"만 청산 → 더블셀 불가
EOD 강제청산 방어evaluate_holdings.py --exclude-strategy 기본값에 limit_up_chase 추가

7데이터 한계 & 리스크 컨트롤
