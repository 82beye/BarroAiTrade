P6. 동일 종목 매 사이클 중복 매수 (4건 양수 가격 추매)

5/19 4건이 DCA(-2%) 아닌 양수 가격에서 1~6분 만에 추매: - 036930: 09:05:36 → 09:06:38 (62초, +1.5%) - 001430: 09:05:24 → 09:11:52 (6분, +1.8%) - 027360: 09:16:16 → 09:17:18 (62초, +1.3%) - 005500: 14:33:18 → 14:34:19 (61초, +0.6%)

원인 추정: _scan_and_buy 의 session_bought.add 누락 또는 pos_store.create_from_order 실패 → 다음 cycle picker 재선정 → 같은 종목 재매수.

fix: 이중 안전망 - recent_buys: dict[str, datetime] process-local 추적 (30분 cooldown) - audit log fallback (recent_buys 비어있을 때 audit 직접 조회)

P7. 진입 직후 노이즈 매도 (6건 모두 10~17분 보유 후 매도)

5/19 6 손실 종목 모두 진입 후 10~17분 만에 breakeven_stop 발동 (peak +2.5% 도달 후 0% 이하 복귀).

fix: - MIN_HOLD_MINUTES: 10 → 15분 (5분 추가 유예) - HARD_SL_PCT = -5.0 — cooldown 안에서도 극한 손실은 즉시 매도

P8. 시장 국면 SIDEWAYS·BULL 분류로 매수 7건 폭주

기존 daemon BEARISH 만 max_buy=1 적용. 5/19 시장 약세지만 picker bias 로 BULL/SIDEWAYS 분류 → max_buy=2 적용 → 7건 매수 → 5건 손실.

fix: - SIDEWAYS도 max_buy=1 (보수 강화) - BEARISH best_pnl > 50k 임계 추가 (강한 신호만) - BULL 만 max_buy=2 유지

P9. P7 부작용 — 강세 종목 익절 회피

P7 단독 적용 시 274090 peak +8.1% 도달했어도 cooldown 15분 안이라 trail 차단 → 종일 표류 → 실제 +309k → 시뮬 -49k.

fix: cooldown 안 매도 정책 분기 - 익절(trailing/take_profit/partial_tp) 통과 - 방어(breakeven/stop_loss -4~-5%) 차단 - hard SL(-5%) 우회 (P7 유지)
