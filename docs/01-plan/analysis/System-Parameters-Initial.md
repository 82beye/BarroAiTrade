## 1. RiskLimits 파라미터

### 현재 기본값 vs 권장값

| 파라미터 | 기본값 | **권장값** | 변경 이유 |
|----------|--------|-----------|-----------|
| `max_position_pct` | 10% | **8%** | 초기 분산도 강화, 단일 종목 쏠림 방지 |
| `max_concurrent_positions` | 5 | **5** | 유지 — 초기 5종목 적절 |
| `max_total_exposure_pct` | 50% | **40%** | 현금 60% 유지로 리스크 버퍼 확보 |
| `stop_loss_pct` | -2% | **-2%** | 유지 — 검증된 손절 기준 |
| `take_profit_1_pct` | +3% | **+2.5%** | 빠른 부분 익절로 수익 확정 |
| `take_profit_1_qty_pct` | 50% | **50%** | 유지 |
| `take_profit_2_pct` | +5% | **+5%** | 유지 — 전량 익절 기준 |
| `daily_loss_limit_pct` | -3% | **-2%** | 엄격한 일일 손실 제한 (모의투자 데이터 신뢰도 향상) |
| `force_close_time` | 14:50 | **15:00** | 장 마감 30분 전 (유동성 충분한 시간) |

### Python 설정 예시

```python
from backend.models.risk import RiskLimits

# 1단계 모의투자 보수적 파라미터
risk_limits = RiskLimits(
    max_position_pct=0.08,          # 종목당 최대 8%
    max_concurrent_positions=5,      # 동시 최대 5종목
    max_total_exposure_pct=0.40,    # 총 투자 40%
    stop_loss_pct=-0.02,            # -2% 손절
    take_profit_1_pct=0.025,        # +2.5% 1차 익절
    take_profit_1_qty_pct=0.50,     # 1차 익절 시 50% 매도
    take_profit_2_pct=0.05,         # +5% 전량
    daily_loss_limit_pct=-0.02,     # -2% 일일 손실 한도
    force_close_time="15:00"        # 강제청산 시간
)
```

## 2. SignalScanner 파라미터

### 현재 기본값 vs 권장값

| 파라미터 | 기본값 | **권장값** | 변경 이유 |
|----------|--------|-----------|-----------|
| `timeframe` | "5m" | **"5m"** | 유지 — 단기 신호 최적 |

## 5. 실행 계획

### 5.1 모의투자 1단계 시작 조건

- [ ] ScannerService에 1차 스캔 대상 20종목 등록
- [ ] RiskLimits 권장값 적용
- [ ] 일별 결과 모니터링 (2주간)
- [ ] 신호 정확도 60% 이상 유지 확인

### 5.2 파라미터 조정 기준

**2주 후 재평가 기준**:

| 조건 | 조치 |
|------|------|
| 신호 정확도 > 65% | 파라미터 유지, 2단계 진행 |
| 신호 정확도 55~65% | 스캔 임계값 상향 검토 |
| 신호 정확도 < 55% | 종목 리스트 재선별 필요 |
| 일일 손실 -2% 도달 빈번 | 손절 기준 재검토 |
