1. 키움 토큰 인증장애 — 원인과 수정 BAR-OPS-31

06-05 장중 키움 토큰 인증실패(rc=3: 8001 App Key/Secret 검증실패 · 8005 Token 무효)가 폭발해 종가 데몬이 매수0/매도0으로 종료. 크레덴셜은 정상이었다.

근본 원인: 키움 자체 OpenAPI는 appkey당 토큰 1개만 유효(새 발급 시 이전 토큰 무효화)+발급 빈도 제한인데, 7개 진입점(API routes·ST 데몬·intraday 데몬·telegram·ohlcv 스크립트)이 공유 캐시 없이 각자 토큰을 발급 → 상호 무효화(8005) 캐스케이드 → 발급 폭주 → 발급 거부(8001). 8001은 자격증명 오류가 아니라 발급 거부의 표면 메시지.

증거: 06-05 08:35:50.122/.142 — 22ms 간격 동시 2건 발급(멀티 프로세스 경합 직접 증거). 토큰 디스크 캐시 없음.
수정: KiwoomNativeOAuth에 파일 캐시+파일락(fcntl) 도입 → 전 프로세스가 단일 토큰 공유·재사용. 동시 8 인스턴스 → 발급 1회로 검증. 0600 권한, 거부 토큰 재채택 차단. (전체 테스트 통과, main 머지 완료)

9. 결론 · 권고

1. 인프라: 토큰 공유캐시로 인증장애 근본 해소. 발급/갱신 알림·백오프 안정화 유지.
2. 전략 배치: 가장 큰 보완은 "최우수 swing_38을 라이브에 켜고, 최약체 supertrend 비중을 억제"하는 배치 자체. swing_38·gold_zone 중심 운용.
3. 청산: supertrend는 청산(트레일/익절) 보강이 손익을 좌우. SELL전환 단독 청산의 지각 문제 해소가 핵심.
4. supertrend: 약전략 — 게이트 강화로 드래그는 줄였으나 out-of-sample 음수. 실거래 비중 최소 또는 추가검증 전 보류 권장.
5. 국면: 하락장에선 분산(종목수 유지), 상승장에선 종목수보다 추세 강도·종목 질에 집중.
